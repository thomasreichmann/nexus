# Issue Audit Checklist

Use this checklist when reviewing AI-drafted issues.

## Description Quality

- [ ] Clear explanation of what needs to be done
- [ ] Explains why this is needed (business value)
- [ ] Technical approach is sound
- [ ] No ambiguous language

## Acceptance Criteria

- [ ] Each criterion is testable (can verify pass/fail)
- [ ] Criteria are specific, not vague
- [ ] Covers all aspects of the feature
- [ ] No missing edge cases
- [ ] Realistic and achievable

## Out of Scope

- [ ] Clearly defines boundaries
- [ ] Reasonable exclusions
- [ ] No important items incorrectly excluded

## Technical Accuracy

- [ ] Approach aligns with codebase patterns
- [ ] Doesn't conflict with existing features
- [ ] Complexity estimate seems reasonable
- [ ] No obvious technical issues

## Completeness

- [ ] Ready for implementation without further clarification
- [ ] All decisions have been made
- [ ] No open questions remain

## Verdict

- [ ] **Approve** - Ready for implementation
- [ ] **Approve with edits** - Minor fixes needed
- [ ] **Send back** - Needs re-grooming
- [ ] **Skip** - Review later
